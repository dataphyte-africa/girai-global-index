# Sanity CMS Integration — Editorial Copy

This document is the implementation guide and checklist for letting the GIRAI client edit
the website's **static editorial copy** from Sanity, with no code changes.

It reflects the agreed architecture decisions (see [Decisions](#1-decisions-locked)). Follow
the phases in order. Phase 0 stands up and **verifies** the full infrastructure on a single
pilot page; Phase 1 then migrates **every** page, seeds the content, and ships in one pass.

---

## 1. Decisions (locked)

| # | Decision | Choice |
|---|----------|--------|
| 1 | **Scope** | Static editorial/marketing copy only — hero, about, methodology, takeaways, header, footer, CTAs, SEO meta, section headings/labels. **Data-driven content** (scores, rankings, country narratives, evidence, indicators) stays generated by the build scripts and is **out of scope**. |
| 2 | **Studio hosting** | Embedded in this Next.js app at `/studio` (one repo, one deploy). Client logs in at `https://<site>/studio`. |
| 3 | **Content modeling** | One **singleton document per page** (Home, About, Methodology, Takeaways) + shared singletons (Header, Footer, Site/SEO settings). Each has typed fields mirroring the page's sections. |
| 4 | **Freshness** | Static generation + **on-demand revalidation** via a Sanity webhook hitting a Next.js route. |
| 5 | **Text** | Mixed: plain `string` for headings/labels/CTAs; **Portable Text** only for body copy needing paragraphs/bold/links/highlighted spans. |
| 6 | **Preview** | Yes — Next.js **Draft Mode** + Sanity **Presentation** tool for live side-by-side preview of unpublished edits. |
| 7 | **Images** | Content images only (team photos, contributor logos, takeaways/feature images, OG image). Decorative assets (`decor1–4`, globe, icons, logos) stay in `/public`. |
| 8 | **Localization** | English only. No i18n — schemas are plain single-language fields. |
| 9 | **Sanity project** | New project created inside the **client's existing Sanity org** (you are already invited). Two datasets: `production` and `development`. |
| 10 | **Migration** | A dev-run **seed script** pushes all current hardcoded copy into Sanity as the initial documents. |
| 11 | **Resilience** | Every section keeps its current hardcoded copy as a **fallback**; Sanity overrides when present. A blank field or Sanity outage never breaks the site. |
| 12 | **Rollout** | **Phase 0:** full infra + verify on one pilot page (About). **Phase 1:** big-bang — all pages wired, seeded, shipped in one pass. |

**Deploy target:** Firebase App Hosting (Cloud Run / Node runtime). ISR, on-demand
revalidation, Draft Mode, and the embedded Studio all work. Secrets go via `apphosting.yaml`
+ Cloud Secret Manager.

## 2. Architecture at a glance

```
Sanity Studio (/studio)  ──edit/publish──>  Sanity Content Lake (dataset: production)
        │                                              │
   Presentation tool                          GROQ reads (useCdn)
   (Draft Mode preview)                                │
        │                                              ▼
        └──────────────>  Next.js (App Router, Server Components on Cloud Run)
                                   │  page.tsx fetches singleton -> props into sections
                                   │  hardcoded fallback if field/doc missing
                                   ▼
                            Static HTML  <──on publish── Sanity webhook -> /api/revalidate
```

Key rules:
- **Server components fetch Sanity** in `page.tsx` and pass copy as props into the existing
  section components. `"use client"` sections never fetch Sanity themselves.
- **Layout/styling stay untouched.** Only text strings, Portable Text bodies, and a few
  content images come from Sanity.
- **Fallbacks live in code** so the migration is safe and incremental.

## 3. Content inventory (editorial copy in scope)

Singletons map 1:1 to pages. Sections listed are the editable units within each.

### Shared singletons
- **Site settings / SEO** (`layout.tsx`): default title, title template, description, keywords, OpenGraph + Twitter title/description, **OG image** (content image).
- **Header** (`header.tsx`): primary nav items, "Explore" mega-menu columns (Explore, Regions, Dimensions, Indicators labels/links), "Download Data" CTA label + href. *(Note: Regions/Dimensions/Indicators link lists are partly data-derived — only the labels and the curated "featured" picks are editorial.)*
- **Footer** (`footer-section.tsx`): Subscribe heading + body + input placeholders + button label; link groups (Results, Explore Regions, Other Projects) labels/hrefs; social links.

### Page singletons
- **Home** (`app/page.tsx`):
  - Hero (`hero-section.tsx`): headline (Portable Text — has highlighted span), subtext.
  - Why GIRAI matters intro (`why-girai-matters-intro-section.tsx`).
  - Dimensions section heading (`dimensions-section.tsx`) — heading/subtitle only; dimension data stays generated.
  - Report download (`report-download-section.tsx`): heading, body, CTA.
  - Top takeaways carousel heading (`top-takeaways-carousel-section.tsx`).
  - Evidence explorer heading + stat labels (`evidence-explorer-section.tsx`) — labels only; numbers stay computed.
  - Choropleth heading (`choropleth-map-section.tsx`).
  - Comparison section heading (`comparison-section.tsx`).
  - Indicator category section copy (`indicator-category-section.tsx`).
  - Limits of measurement (`limits-of-measurement-section.tsx`).
  - Our impact (`our-impact-section.tsx`).
  - Shaping intelligence (`shaping-intelligence-section.tsx`).
- **About** (`app/about/page.tsx`) — **PILOT**: Hero, Intro, Why GIRAI Matters, GCG-led, What the Index Measures, Contributors, Who GIRAI Is For, People Behind Research (photos = content images), Footer hero.
- **Methodology** (`app/methodology/page.tsx`): Hero, Intro, Principles, What it Measures, Framework Evolution, Framework Refined, Edition Changes, Key Terms, Evidence Standards, Access Data, Research Process, Footer hero.
- **Takeaways** (`app/takeaways/page.tsx`): Hero, Intro, Key Insights. *(The takeaways list itself is data-driven — out of scope.)*
- **Indicators** (`app/indicators/page.tsx`): Hero (title, lead, background image).
- **Evidence Explorer** (`app/evidence/page.tsx`): Hero (title, subtitle, stat labels), pathway picker heading/subtitle. *(Stat values and evidence rows stay generated.)*
- **Regions** (`app/regions/page.tsx`): Hero, performance overview heading/subtitle, compare section heading/subheading.
- **Countries** (`app/countries/page.tsx`): Hero, compare section heading/subheading, top takeaways section header. *(Takeaway accordion items stay hardcoded.)*

> Detail routes (`/indicators/[slug]`, `/regions/[slug]`, `/countries/[iso3]`, `/evidence/[itemId]`, `/dimensions/[slug]`) and data-heavy sections (scores, rankings, choropleth, indicator lists) remain **out of scope** — generated from the dataset pipeline.

## 4. Environment variables

```env
# .env.local (development) — committed values are non-secret except the token
NEXT_PUBLIC_SANITY_PROJECT_ID=<project_id>
NEXT_PUBLIC_SANITY_DATASET=development        # production in prod
NEXT_PUBLIC_SANITY_API_VERSION=2024-10-01

# Server-only secret (Viewer token for Draft Mode/preview + seed script writes)
SANITY_API_READ_TOKEN=<viewer_token>
SANITY_API_WRITE_TOKEN=<editor_token>         # used ONLY by the seed script, not the app
SANITY_REVALIDATE_SECRET=<random_string>      # shared with the Sanity webhook
```

On **Firebase App Hosting**, expose the public ones as build+runtime env in `apphosting.yaml`,
and store `SANITY_API_READ_TOKEN` + `SANITY_REVALIDATE_SECRET` in **Cloud Secret Manager**,
referenced from `apphosting.yaml`. The **write token is never deployed** — it's only used
locally/CI by the seed script.

## 9. Risks & notes

- **Client components** (`"use client"` sections) must receive copy as props from a server
  parent; they never import the Sanity client.
- **Tokens**: only the public project ID/dataset reach the browser. Viewer/Editor/revalidate
  secrets are server-only; the write token is local/CI-only.
- **Header/Footer link lists** are partly derived from data (`DIMENSIONS`, `INDICATORS`,
  regions). Only labels and curated "featured" entries are editorial — keep the derived parts
  in code to avoid the client breaking navigation.
- **Fallbacks are the safety net**: ship every section with code defaults so a missing field
  or Sanity outage degrades gracefully.
- **Out of scope**: anything produced by `scripts/build-data.ts` /
  `build-narratives.ts` and the generated JSON under `src/data/2026/generated/` and
  `public/data/`.
